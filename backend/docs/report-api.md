# Report API

API для веб-интерфейса начальника. Все запросы идут через Traefik.

## Авторизация

Нужен токен начальника:

```http
Authorization: Bearer dev-admin-token
```

Работник с `Bearer dev-token` получает `403 Forbidden`.

## Сводка

```http
GET /api/reports/analytics/summary
```

Назначение: верхние показатели для дашборда начальника.

Возвращает:

- количество обходов по статусам;
- количество открытых дефектов;
- количество предупреждений и критических отклонений;
- средний процент заполнения чек-листов.

## Отчёты по обходам

```http
GET /api/reports/rounds
```

Query params:

- `status` - фильтр по статусу обхода;
- `employee_id` - фильтр по работнику;
- `limit` - размер страницы, по умолчанию `50`;
- `offset` - смещение, по умолчанию `0`.

Назначение: список обходов для таблицы отчётов.

## Детальный отчёт обхода

```http
GET /api/reports/rounds/{round_id}
```

Назначение: карточка конкретного обхода.

Возвращает:

- общую информацию по обходу;
- заполненные пункты чек-листа;
- показания оборудования;
- выявленные дефекты.

## Аналитика по оборудованию

```http
GET /api/reports/analytics/equipment
```

Query params:

- `limit` - количество записей, по умолчанию `20`.

Назначение: показать оборудование, которое чаще попадает в отклонения или дефекты.

## Аналитика по работникам

```http
GET /api/reports/analytics/employees
```

Query params:

- `limit` - количество записей, по умолчанию `20`.

Назначение: показать выполнение обходов по сотрудникам: количество обходов, подтверждённых точек, среднее время выполнения и найденные отклонения.
